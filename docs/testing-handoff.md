## Safety Scope

The test additions are scoped to compiler correctness and deterministic runtime
semantics.

Allowed and implemented:

- Build the compiler.
- Tokenize and parse safe Entropia source snippets.
- Type-check safe source snippets.
- Compile safe source snippets into shellcode or BOF artifacts.
- Inspect artifact shape, such as non-empty raw shellcode and x86-64 COFF
  headers.
- Execute local zero-argument shellcode tests on Linux x86-64 to verify
  deterministic CPU return values.

Not implemented in this pass:

- EDR bypass validation.
- AMSI or ETW patch validation.
- Sleep-mask bypass detection tests.
- Direct or indirect syscall evasion runtime checks.
- LSASS dumping, token theft, injection, or offensive runtime behavior.
- BOF execution on Debian.

Reason: those items test operational evasion or Windows-only behavior rather
than portable compiler correctness. BOF execution in this repo currently routes
through Windows-only loader code that depends on `VirtualAlloc`, Win32 import
resolution, and Beacon-style calling conventions.

Purpose:

- Move beyond static artifact checks.
- Verify that generated shellcode produces correct CPU-observed return values.
- Keep the tests local, deterministic, and Linux x86-64 only.

Runtime harness added inside `compiler_gauntlet_tests`:

```rust
#[cfg(all(unix, target_arch = "x86_64"))]
unsafe fn run_shellcode_i64(code: &[u8]) -> i64
```

Harness behavior:

- Allocates executable memory with `mmap`.
- Uses `PROT_READ | PROT_WRITE | PROT_EXEC`.
- Copies generated shellcode bytes into the mapping.
- Calls the mapping as `extern "win64" fn() -> i64`.
- Releases memory with `munmap`.

Reason for `extern "win64"`:

- Entropia emits Windows x86-64 calling convention code.
- The existing `run_bin.rs` uses the same approach for zero-argument shellcode
  on Linux x86-64.

## Known Issues

### cargo fmt

```bash
cargo fmt --all -- --check
```

```text
fails
```

Reason:

- The repository has broad pre-existing rustfmt drift.
- The first reported diff starts at `src/asm.rs`.
- Running `cargo fmt --all` creates a very large unrelated formatting diff
  across many Rust files.

Decision taken:

- Do not run broad formatting as part of the compiler test additions.
- Treat fmt failure as a known legacy formatting issue until a dedicated
  formatting reset is requested.

The build and tests pass but emit existing warnings, including:

- `with_gc_mode` is never used.
- several COFF constants are never used.
- `typecheck::check` is unused in release builds.
- `entc-debug` has unused/dead-code warnings under tests.

These warnings were not fixed in the testing work because they are not required
for compiler gauntlet coverage.
